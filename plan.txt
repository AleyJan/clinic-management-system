Think of this project like a hospital building. Before you build rooms, you need:

The foundation (Auth + DB)
The skeleton (Routes + Roles)
The rooms (Features per role)
The smart systems (AI layer)

🧱 Component Independence Strategy
Here's how each piece stays isolated:
src/
├── pages/
│   ├── admin/          ← crashes here? Doctor still works
│   ├── doctor/         ← crashes here? Patient still works
│   ├── receptionist/
│   └── patient/
├── components/
│   ├── shared/         ← Navbar, Sidebar, Loader (used by all)
│   ├── appointments/   ← AppointmentCard, BookingForm
│   ├── prescriptions/  ← PrescriptionForm, PDFDownload
│   ├── ai/             ← SymptomChecker, RiskFlag (optional layer)
│   └── analytics/      ← Charts (only Admin/Doctor see these)
├── context/
│   └── AuthContext.jsx ← global user/role state
├── api/
│   └── axios.js        ← one central API file
Each folder is a self-contained world. A bug in ai/ never touches appointments/.
